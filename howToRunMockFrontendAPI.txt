# CrisisCloud (Flask + Leaflet + NWS Weather API)

CrisisCloud is a Flask web application that serves a Leaflet-based frontend and pulls live weather, weekly forecast data, and active alerts from the National Weather Service (`weather.gov` / `api.weather.gov`) based on the current map center.

This version of the application does not use a database. Resource data is stored in the Python backend as shared in-memory demo data, and weather data is retrieved live from the National Weather Service API.

## Current Features

- Live map interface built with Leaflet
- Real-time weather and active alerts from the National Weather Service
- Weekly forecast display for the current Hampton Roads map area
- Shared backend resource feed for:
  - Storm shelters
  - Food banks
  - Police departments
  - Fire services
  - Towing services
- Organization login panel for updating shared resource status during demos
- Sidebar directories for:
  - Emergency contacts
  - Shelter and food bank listings
- Three manually placed real shelter locations currently loaded in the backend:
  - Blair Middle School Shelter
  - Granby High School Shelter
  - Norview High School Shelter

## Prerequisites

- `pip`

Make sure your files are organized like this:

```text

├── README.md
```

## Install Dependencies

From a terminal in the project folder, run:

```powershell
pip install flask requests python-dotenv
```

## Environment Variables

Create a file named `.env` in the same directory as `crisisCloudBackend.py` with:

```env
NWS_USER_AGENT=CrisisCloud/1.0 (your-email@school.edu)
```

This User-Agent is required by the National Weather Service.

Do not place the `.env` file inside the `templates` folder.

## How Shared Resource Data Works

- The app currently stores shelter, food bank, police, fire, and towing entries directly inside `crisisCloudBackend.py`.
- These entries are served by:
  - `GET /api/resources`
  - `POST /api/resources/<resource_id>`
- Updates made through the Organization Portal are shared across open browser tabs while the Flask app is running.
- Because this is in-memory data, restarting the Flask server resets resources back to the values in `crisisCloudBackend.py`.

## Manual Shelter Coordinates

The current real shelter locations are manually stored in `crisisCloudBackend.py` using known latitude and longitude values.

To add more real shelters later:

1. Find the school or shelter address.
2. Convert the address to latitude and longitude.
3. Add a new shelter entry in `crisisCloudBackend.py`.
4. Restart the Flask server.

## Common Issues

### `TemplateNotFound`

Make sure `crisisCloud.html` is inside the `templates` folder and that the Flask route references:

```python
return render_template("crisisCloud.html")
```

### Weather Not Loading

- Confirm the frontend is calling `/api/weather/live`
- Confirm your `.env` file exists and includes `NWS_USER_AGENT`
- Check that the machine running Flask has internet access

### `.env` Not Loading

- Ensure the file is named exactly `.env`
- Keep it in the project root, not in `templates`
- Restart Flask after editing it

### Resource Updates Reset After Restart

This is expected in the current version. Resource updates are stored in memory only and are reset whenever the Flask server restarts.
