# CrisisCloud (Flask + Leaflet + SQLite + NWS Weather API)

CrisisCloud is a Flask web application that serves a Leaflet-based frontend and pulls live weather, weekly forecast data, and active alerts from the National Weather Service (`weather.gov` / `api.weather.gov`) based on the current map center.

This version of the application uses a local SQLite database file named `crisiscloud.db` to store shelters, food banks, police stations, fire stations, hospitals, and towing services. Weather data is still retrieved live from the National Weather Service API.

## Current Features

- Live Leaflet map with custom resource icons
- Live weather, alerts, and weekly forecast from the National Weather Service
- SQLite-backed shared resource data
- Real-world locations for shelters, food banks, police, fire, hospitals, and towing services
- City filtering for:
  - All
  - Norfolk
  - Chesapeake
  - Portsmouth
  - Suffolk
  - Virginia Beach
- Organization login panel for updating resource status, phone number, and availability
- Emergency Contact Directory split into:
  - Police Stations
  - Fire Stations
  - Hospitals
  - Towing Services
- Separate Storm Shelter Directory and Food Bank Directory
- Collapsible directory sections
- Click-to-focus behavior that scrolls to the map and shows the selected location
- Local network hosting support for two-computer demos

## Install Dependencies

From a terminal in the project folder, run:

```powershell
pip install flask requests python-dotenv
```

## Environment Variables

Create a file named `.env` in the same directory as `crisisCloudBackend.py` with:

```env
NWS_USER_AGENT=CrisisCloud/1.0 (your-email@school.edu)
```

This User-Agent is required by the National Weather Service.

Do not place the `.env` file inside the `templates` folder.

## Run Across Two Computers On The Same Network

The Flask app is configured to listen on the local network.

When you start the app, the terminal should show something like:

```text
Running on http://127.0.0.1:5000
Running on http://192.168.x.x:5000
```

To test on a second computer:

1. Keep the app running on the host computer.
2. Make sure both computers are on the same Wi-Fi or local network.
3. On the second computer, open the network address shown by Flask, for example:

```text
http://192.168.0.159:5000
```

Both computers will then connect to the same running backend and shared `crisiscloud.db` file.

## How Resource Data Works Now

- The app serves resource data through:
  - `GET /api/resources`
  - `POST /api/resources/<resource_id>`
- The frontend polls the backend so updates can appear after refresh or during the demo polling cycle.
- Resource updates are now stored in SQLite, so they persist after restarting the app.
- The backend will seed the database automatically if the database starts empty.

## How To Share The Latest Changes With The Team

The easiest way is to share the latest version of:

```text
crisiscloud.db
```

If a teammate replaces their local `crisiscloud.db` with the newest one, they will see the latest city data the next time they run the app.

## Adding More Real Locations

To add more real locations later:

1. Get the real address.
2. Find the latitude and longitude.
3. Add the record to the backend seed list in `crisisCloudBackend.py`.
4. Insert or update the record in `crisiscloud.db`.
5. Restart the app or refresh the page.

## Common Issues

### `TemplateNotFound`

Make sure `crisisCloud.html` is inside the `templates` folder and that the Flask route references:

```python
return render_template("crisisCloud.html")
```

### Weather Not Loading

- Confirm the frontend is calling `/api/weather/live`
- Confirm your `.env` file exists and includes `NWS_USER_AGENT`
- Check that the machine running Flask has internet access

### `.env` Not Loading

- Ensure the file is named exactly `.env`
- Keep it in the project root, not in `templates`
- Restart Flask after editing it

### Database Looks Out Of Date

If a teammate is missing newer city records, make sure they have the latest copy of:

```text
crisiscloud.db
```

### Second Computer Cannot Connect

- Confirm both computers are on the same network
- Confirm the host computer is still running Flask
- Confirm Windows Firewall allows Python on private networks
- Use the host machine's network URL shown in the Flask terminal output
